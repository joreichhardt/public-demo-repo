# Schreib-Roboter Demo-Repo

Dieses Repo zeigt die Arbeitsweise aus dem Buch in einer kleinen, öffentlichen Form.
Es enthält nur kuratiertes Beispielmaterial:

- Markdown statt proprietärem Datei-Format
- Git-Historie statt flüchtigem Chatverlauf
- klare Ordner für Manuskript, Planung, Agenten und Export
- keine sensiblen Notizen, keine privaten Quellen, keine Rohfassungen

Ziel ist nicht ein komplettes Buch, sondern ein nachvollziehbares Muster für einen
offenen Schreibworkflow.

## Inhalt

- `AGENTS.md`: Regeln für die Zusammenarbeit mit dem Coding Agent
- `planung/`: Ziel, Abgrenzung und Kapitelidee
- `manuskript/`: ein kurzer Beispieltext
- `export/`: Hinweise für EPUB- und Druckausgabe
- `scripts/`: Platz für kleine Build-Helfer

## Lesereihenfolge

1. `planung/expose.md`
2. `planung/kapitelplan.md`
3. `AGENTS.md`
4. `manuskript/kapitel-01.md`

## Wofür das Repo steht

Dieses Demo-Repo zeigt die Kernidee des Buchs:

- Die Dateien gehören dem Projekt.
- Der Agent arbeitet im Projekt, nicht in einem geschlossenen Format.
- Export und Veröffentlichung sind letzte, technische Schritte.

## Vor der Veröffentlichung

- Ein klares Lizenzfile ergänzen.
- Echte Exportdateien nur dann hinzufügen, wenn sie öffentlich sein dürfen.
- Sensible Quellen, Rohentwürfe und private Notizen draußen lassen.
